# 第三章 Obsidian+Git本地云端同步存储
## 3.1 官网下载obsidian和git
方法一：插件Git，可以自动提交同步
方法二：在本地文件夹目录下输入cmd
```bash
git add .                               //把该目录下的所有文件都添加到箱子里
git commit -m "新增了JavaScript修炼文件"  //-m是告诉git这次提交的注释，必须有，方便回溯
git push origin JavaScript              //origin是远程仓库代号，JavaScript是当前分支

补充：
git init   把普通的文件夹变为受git保护的文件夹
git status 可以看当前的状态（发生了哪些变化）


如果连接github失败
#### ⚠️ 网络连接排错 (Port 443)
1. **排查端口**：通过 `系统设置 > 代理` 查看。
2. **确认监听**：使用 `netstat -ano | findstr "LISTENING"` 确认端口号（如 10810）是否活动。
3. **配置 Git**：
   - 开启代理：`git config --global http.proxy http://127.0.0.1:10810`
   - 关闭代理：`git config --global --unset http.proxy`
```

## 3.3补充知识
- 127.0.0.1代表本机，别名为localhost
- TCP（可靠，类似打电话，必须你接ta才会打），UDP（低延迟，发传单，只负责发）是传输协议，
- 端口，一台电脑有很多端口，把ip想像为一个房子，那每个端口就是房子里的一个房间。每个网络软件在在运行时，都会在电脑占用一个特定的端口号
```bash
知名端口（绝对固定，全球统一，不可更改，0-1023）：
sql server（1433）；http（80），https(443)，这个是服务器的端口

注册端口（相对固定，软件公司为自己在电脑上申请的一个固定办公位,1024-49151）：
clash verge（7897,10810）,live server插件（5500）解释：为了让你方便找到它，通常会申请一个固定端口，这样你在浏览器输入 `127.0.0.1:5500` 就能看到网页

随机端口（49152-65535）：
当打开一个软件，操作系统会随机分配一个端口（有空房间就进去）

	netstat -ano | findstr "LISTENING"  //查看活动端口
```
